Getting Started

To use this plugin you need :
  • A database (started and ready to accept connections)
  • A project in the Eclipse workspace (Java project or other)
  • The jar file containing the JDBC driver for database connection.
For a full documentation see the web site : https://sites.google.com/site/telosystools/

The main steps are described bellow :

Configure the project

Right-click at the root of the project and select "Properties" - "Telosys Tools"

In the "Telosys Tools" properties windows you can ...
  • Define the project's folders
  • Initialize the environment by creating the required folders and files
  • Download predefined templates from GitHub
  • Define the specific variables used in your templates
  • etc

Configure the database connection and generate the repository

Before trying to connect to a database you must have the JDBC driver.
The JAR file can be located either in the Telosys Tools library folder (usually "TelosysTools/lib")
or in the project class path if it's a Java project.

To define a new database, open the "databases.dbcfg"
Just double-click on the file to open the databases configuration editor.
Here you can :
. Define a new database with its JDBC configuration
. Test the connection
. Check that all the expected meta-data can be retrieved

When everything works correctly, save the file.

Finally you can generate a new repository (or later update an existing repository).

Customize the repository

Open the generated repository (".dbrep" file)
and customize it if necessary with the editor

Get templates and launch generation

Right-click at the root of the project and select "Properties" - "Telosys Tools"
Select the "Download" tab
Get the templates available on GitHub and download the templates of your choice
Copy the donwloaded templates in the "templates" folder

Re-open the repository file (.dbrep) or refresh the targets list with the "Refresh" button
The list of the available targets is defined in the "templates.cfg" file (it can be edited from the repository editor)

Once all the templates are ready you can generate the targets entity by entity
or use the "bulk generation" page to launch a batch generation for multiple entities

Create your own targets/templates

To create a new generation target (a new kind of file to be generated)
  • Create a new template file (usually copy an existing .vm file and change it)
  • Declare the new target with its template and destination folder in "templates.cfg"
  • Refresh the targets list and use it...
